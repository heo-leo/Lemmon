# v1 — 모니터 분리/재연결 시 창 위치 복원 신뢰성 개선

## Problem

3-디스플레이(맥북 + 외장 모니터 2대) 환경에서 두 외장 모니터를 분리했다가
재연결하면, 음수 x 좌표 영역(맥북 기준 좌측)에 있던 창들이 보조 모니터로
복원되지 못하고 맥북 화면으로 모이거나 모니터 가장자리에만 살짝 걸치는
이슈. 우측 모니터(양수 x) 창들은 비교적 정상 복원됨.

## Root causes

진단 빌드의 NSLog 트레이스로 다음 4가지 결함이 누적되어 일어남을 확인:

1. **`saveState` 시점 — `applicationDidChangeScreenParameters`는 macOS가
   이미 창들을 새 레이아웃에 맞춰 자동 이동/축소시킨 *뒤*에 발화**한다.
   따라서 그 시점에 `getState()`로 읽은 좌표는 “원본”이 아니라 “재배치 후”
   값이고, 이것이 `state[oldNumScreens]`로 들어가 다음 재연결 시 손상된
   좌표가 그대로 복원됨.

2. **`saveState`의 비-current `kNum` 슬롯이 “현재 안 보이는 앱/창”을
   드롭**한다. 분리 직후 일부 앱이 사라지는 디스플레이의 창을 일시적으로
   숨기면, 다음 saveState 호출에서 그 앱의 saved 엔트리가 통째로 사라짐.

3. **`setWindowSizes`의 카운트 가드 (`win.count == sizes.count`) 가 너무
   엄격**하다. `kAXWindowsAttribute`는 보조/숨김 창까지 포함해 반환하기
   때문에 (예: Chrome은 가시 창 2개에 대해 AX 22개), CGWindowList 기반
   saved 엔트리와 인덱스로 매칭하면 카운트 불일치 시 *해당 앱 전체가 SKIP*
   되어 그 앱의 창들이 macOS가 옮긴 위치(맥북) 그대로 남음. 이번 케이스의
   실제 “좌측 → 맥북” 증상의 직접 원인.

4. **`setWindowSizes`의 AX 호출 순서**(position → size)는 일부 앱에서
   AX 클램핑/리사이즈 충돌을 일으켜 가장자리 걸침을 만들 수 있음.

또한 재연결은 자주 단계적(1 → 2 → 3)으로 일어나며, 중간 단계에서는
`NSScreen.screens`의 좌표가 임시 위치/해상도로 잡혀 있어 AX가 saved
좌표를 거부/클램핑하는 케이스도 관측됨.

## Fixes (모두 `src/main.swift` 한 파일)

### F1. 변경 직전 스냅샷 — `CGDisplayRegisterReconfigurationCallback`

`kCGDisplayBeginConfigurationFlag` 단계(디스플레이 변경 *직전*)에서
`getState()`를 호출해 `preChangeSnapshot`/`preChangeNumScreens`에 보관.
`applicationDidChangeScreenParameters`에서는 이 사전 스냅샷을
`saveState`로 넘겨 사용 — macOS의 자동 재배치가 끼어들기 전 좌표가
`state[oldNumScreens]`에 들어가도록 보장.

### F2. `saveState`의 보존 로직 강화

`saveState`에 `snapshot`/`snapshotNumScreens` 옵션 인자 추가. 비-current
`kNum`은 기존 `state[kNum]`을 시작점으로 삼아, `newState`에 보이지 않는
앱/창의 saved 엔트리를 절대 드롭하지 않도록 변경 (`seen` Set로 추적해
누락된 winNum 엔트리를 그대로 끝에 보존).

### F3. AX ↔ CGWindow 매칭을 winNum 기반으로

비공개 SPI `_AXUIElementGetWindow(axElem, &cgID)` 를 도입하고
`setWindowSizes`를 다음과 같이 재작성:

- `axWinList`로 받은 AXUIElement 각각에 대해 CGWindowID 조회.
- `axByWin: [WinNum: AXUIElement]` 맵 구성.
- saved `[(winNum, rect)]`를 순회하며 매칭되는 AX 엘리먼트만 골라 적용.

→ 카운트 가드 제거. AX가 22개 반환해도 saved 2개에 해당하는 정확한 두
창만 이동되고 나머지는 손대지 않음.

### F4. AX 호출 순서를 size → position → size → position 으로

먼저 사이즈를 줄여 AX가 새 origin을 “현재(어쩌면 더 큰) 프레임 기준”
으로 클램핑하지 않도록 한 뒤 위치를 잡고, 정확한 사이즈로 다시 맞추고,
사이즈 변경이 위치를 흔드는 일부 앱 케이스를 마지막 position 재확정으로
마감.

### F5. 재연결 후 지연 retry

재연결은 보통 단계적으로 일어나며 중간 상태의 NSScreen 프레임은 임시
위치/해상도일 수 있음. `applicationDidChangeScreenParameters` 끝에
`scheduleRetry(after: 0.4)` 와 `1.5` 두 차례를 예약해, 모든 디스플레이가
정착한 뒤 한 번 더 `restoreState`를 호출. retry는 `spacesNeedRestore`를
재충전하고 idempotent하게 동작.

## Verification

- 진단 빌드로 NSLog 트레이스 수집 → 위 F1–F4 가 각각 누락/잘못된
  좌표/SKIP/클램핑을 만들고 있음을 직접 확인.
- 실 시나리오(외장 4K 2대 동시 분리/재연결)에서 좌·우 모니터 모두 원래
  위치/크기로 복원됨을 사용자가 확인.
- 빌드는 `make CONFIG=debug` 정상, universal 바이너리 ad-hoc 서명 통과.

## 영향 범위

- 단일 파일 정책 유지 (`src/main.swift` 약 325줄).
- 외부 의존성 없음 (`_AXUIElementGetWindow`는 비공개 SPI지만 모든
  주요 macOS 윈도우 매니저가 사용해 온 사실상 안정적인 심볼).
- AX 권한 없을 때의 silent no-op 동작은 그대로.
- 본 변경에 따른 Info.plist/Makefile 수정 없음. 버전 표기(`v1.5`)는
  릴리스 시 별도로 bump.
